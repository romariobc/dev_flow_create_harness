1. A escada de três camadas Milvus

Prompt engineering, context engineering e harness engineering não competem entre si — são uma progressão. Cada camada amplia o alcance da anterior.

Camada 1

Prompt engineering

O que você diz ao modelo

Escopo: uma única troca (um turno). Otimiza o conteúdo da pergunta — few-shot, chain-of-thought, formatação. Não tem alcance sobre o que acontece na sessão seguinte.
→
Camada 2

Context engineering

O que o modelo consegue ver

Escopo: a janela de contexto. Otimiza retrieval, compressão de histórico, o que entra e o que é descartado do contexto disponível.
→
Camada 3

Harness engineering

O mundo em que o agente opera

Escopo: execução autônoma de horas, várias sessões. Ferramentas, lógica de validação, restrições arquiteturais, ciclo de vida da sessão. É aqui que "concluído" passa a significar algo verificável.

3. Evolução multi-agente Anthropic

Um único agente que se autoavalia tende a ser tendencioso a favor do próprio trabalho. A correção, inspirada em GANs, é separar arquiteturalmente quem gera de quem avalia.

Solo

1 agente faz tudo: escreve, decide se está bom, declara concluído

Dois agentes

Initializer — configura ambiente, feature list, git
Coding agent — avança incrementalmente sessão após sessão

Três agentes

Planner — especifica o produto
Generator — implementa (assina "contrato de sprint")
Evaluator — testa com Playwright como um usuário real

Experimento controlado (mesmo modelo, mesma tarefa — "construa um editor de jogo retrô 2D"):

US$ 9
sem harness, 20 min — quebrado
US$ 200
harness de 3 agentes, 6h — jogável
